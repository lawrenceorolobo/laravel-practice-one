Overview

This document covers everything needed to take Quizly from development to production. It breaks down each service, why it's needed, what it costs, and the full deployment sequence. The stack is built around Namecheap for domain, hosting, and email — keeping everything under one roof for simpler management.

Why not shared hosting? Quizly uses WebSockets (Laravel Reverb) for real-time updates — when a candidate submits a test, the dashboard updates instantly. Shared hosting blocks the port binding that WebSockets require. A VPS is the minimum requirement for this feature to work, and it's actually cheaper than most shared plans when you factor in what you get.

Deployment Flow

Here's the exact sequence from "ready to go" to "platform is live." Timeline: 2–3 working days.

1

Domain & DNS Setup — Day 1

  • Purchase domain on Namecheap
  • Point DNS A record to VPS server IP
  • Configure email DNS records (SPF, DKIM, DMARC) for deliverability
2

Server Provisioning — Day 1

  • Create Namecheap VPS (Ubuntu 24.04 LTS)
  • Install stack: PHP 8.3, Nginx, Composer, Node.js, MySQL 8, Supervisor
  • Set up firewall: only ports 80, 443, 22, 6001 open
  • Create deploy user with SSH key authentication
  • Harden server (disable root login, fail2ban)
3

Database Setup — Day 1

  • Install and configure MySQL 8 on the VPS
  • Create production database and restricted user
  • Run all migrations and seed initial data
  • Configure automated daily database backups
4

Application Deployment — Day 1–2

  • Clone repository to server
  • Install production dependencies
  • Configure environment (database, email, Cloudinary, Reverb)
  • Build frontend assets
  • Set file permissions and ownership
5

SSL & Web Server — Day 2

  • Install Certbot and issue Let's Encrypt SSL certificate
  • Configure Nginx with SSL, HTTP→HTTPS redirect, security headers
  • Set up auto-renewal cron for SSL certificate
  • Configure WebSocket proxy (wss:// via Nginx)
6

Background Services — Day 2

  • Configure Supervisor for queue workers (handles email sending, broadcasting)
  • Configure Supervisor for Reverb WebSocket server
  • Set up Laravel task scheduler via cron
  • Verify all services auto-start on reboot
7

Testing & Go-Live — Day 2–3

  • Test full user flow: registration, login, create assessment
  • Test email delivery: invitations, results, password reset
  • Test proctoring: webcam recording and Cloudinary upload
  • Test real-time updates: WebSocket connectivity across pages
  • Monitor server logs for 24 hours
  • Share the live URL
